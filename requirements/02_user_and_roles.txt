Define the following entities with the following attributes and relations.
Only implement the entities, without DTO, repos, services and controllers.

User (table = users)
* id - generated value
* username - max length 20, unique
* email - max length 50, unique
* password - max length 20

Role (table = roles)
* id - generated value
* name - enum as string AppRole (User, Seller and Admin)

Address (table = addresses)
* id - generated value
* street_line_1 - min length 5
* street_line_2 - optional
* city - min length 5
* state - min length 5
* country - min length 2
* zipcode

Users and Roles
* Users has a Many-to-Many relationship with roles (join table = user_role, join column = user_id, role_id)
* fetch = eager, cascade persists and merge

Users and Products
* A user can have many products (cascade persists and merge, remove orphans
* A product has a many-to-one relation with user (name = seller_id)

Users and Addresses
* A user can have many addresses (mapped by = user, cascade persists and merge)
* An address belongs to one user (name = user_id)
